008 - CAMPO DE TAMANHO FIXO - INFORMAÇÃO GERAL (NR)

Este campo contem 40 posições de caracteres (00-39) que contém informação codificada sobre o registro como um todo e sobre aspectos bibliográficos especiais do ítem que está sendo catalogado. Estes dados codificados são potencialmente úteis para recuperação e propósitos administrativos.
MARC21 LITE não usa todos os elementos definidos em cada campo 008; recomenda-se usar o caráter de preenchimento (|). A definição de todos os elementos 008 estão no: Formato MARC 21 Conciso.
Os elementos de dados estão definidos pela sua posição. As posições dos caracteres que não estão definidos contem um espaço em branco (#). Todas as posições de caracteres definidos devem conter um código definido; para algumas posições do campo 008, pode ser o caráter de preenchimento (|). O caráter de preenchimento pode ser usado (em certas posições) quando a organização catalogadora não tenta codificar o caráter desta posição. Os caracteres de preenchimento não são permitidos nas posições do campo 008 00-05 (Data de inclusão no arquivo). Recomenda-se não usá-los nas posições 07-10 (Data 1), 15-17 (Lugar de publicação, produção, ou execução), e na posição 008 definida para Formado ítem (seja a posição 23 ou 29, dependendo da configuração do 008). As posições de caracteres 00-17 e 35-39 são iguais para todos os tipos de materiais, com  consideração especial da posição 006 (Tipo de data de publicação). A definição das posições de caracteres 18-34 foi feita independentemente para cada tipo de material, ainda que certos elementos de dados tem definição igual nas especificações para mais de um tipo de material. Quando elementos de dados similares são definidos para inclusão no campo 008 para diferentes tipos de materiais, estes ocupam as mesmas posições de caracteres no campo 008.
Na documentação que segue, as posições de caracteres são descritos na ordem da lista que está no início desta seção. As posições do campo 008 00-17 e 35-39 para todo tipo de material (Todos os materiais) são descritos primeiro, seguidas por sete grupos de seções para as posições 18-34 (Livros, Arquivos de Computador, Mapas, Música, Séries, Materiais Visuais, e Materiais mistos).

008 - TODOS OS MATERIAIS - POSIÇÕES COMUNS (NR)

Posição dos caracteres
  • 00-05 — Data de inclusão
    Uma cadeia de seis caracteres numéricos gerada pelo computador que contem a data criação do registro. O padrão da data é: aammdd.
  • 06 — Tipo de data da publicação
    Um caracter que indica a que categoria pertence a data informada em 008/07-10 (Data 1) e 008/11-14 (Data 2). Para os documentos tipo série, o código de 008/06 também indica o estado da publicação. A seleção de 008/06 se realiza determinando a data correta para 008/07-14 (Data 2). Para a maioria dos registros esta informação provem da data contida no campo 260 (Publicação, distribuição, etc(Imprenta)), campo 362 (Data de publicação e/ou designação de seqüência), ou de campos de notas.
  • c = Publicação seriada corrente
    As posições 008/07-10 contem a data inicial de publicação;  e as posições 008/07-10 contem os caracteres 9999.
  • d = Publicação seriada  encerrada
    As posições 008/07-10 contem a data inicial de publicação; e 008/11-14 contem a data final de publicação.
  • m = Datas múltiplas
    008/07-10 tem a data inicial e 008/11-14 a data final.
  • r = Data de Reimpressão/reedição e original
    008/07-10 contem data de reprodução ou reedição; 008/11 -14 contema data original se for conhecida; se for desconhecida contem o código u ("uuuu").
  • s = Uma única data conhecida/data provável
  • 008/07-10 contem a data; 008/11-14 contem brancos (####).
  • t = Data de publicação e copyright
  • | = Não se tentou codificar

    07-10 — Data 1
    A determinação das datas para 008/07-10 é feita no mesmo tempo que a seleção do código para 008/006 (tipo de data/estado da publicação). O uso de caracteres de preenchimento em 008/07-10, no obstante seja possível, não se aconselha visto que a data em Data 1 é usada para recuperação e verificação de duplicatas em alguns sistemas. Quando se usa caracteres de preenchimento em 008/07-10, as quatro posições devem conter o caráter de preenchimento.

    1-9 - Dígitos da data

    # - O elemento data não é aplicável

    u - O elemento data é total o parcialmente desconhecido

    |||| - Não se tentou codificar

     

    11-14 — Data 2
    A determinação das datas para 008/11-14 se faz simultaneamente com a seleção do código 008/06 (Tipo de data/estado da publicação). Ver a seção 008/06 com exemplos e convensões de entrada relacionadas para o código de informação de data. Quando os caracteres de preenchimento (||||) são usados em 008/11-14 significa que  nenhuma tentativa foi feita para codificar as posições de caracteres.

    1-9 - Dígitos da data

    # - O elemento data não é aplicável

    u - O elemento data é total o parcialmente desconhecido

    |||| - Não se tentou codificar

     

    15-17 — Lugar de publicação, produção ou execução
    Um código MARC de dois o três caracteres que indica o lugar de publicação, produção ou execução. O código de lugar é um elemento de autoridade da agência catalogadora. A fonte do código é a     Lista de Código MARC por Países. A seleção de um código MARC geralmente está relacionada com a informação do campo 260 (Publicação, Distribuição, etc. (Imprenta)). O armazenamento do código em 008/15-17 e usado em conjunto com o campo 044 (Código do País produtor) quando mais de um código é apropriado no item. O primeiro código no subcampo ^a do campo 044 é armazenado no campo em 008/15-17. Podem ser usados três caracteres de preenchimento (|||) no lugar de um código válido, porem não se recomenda seu uso em 008/15-17. O dado neste elemento do campo 008 é usado para recuperação e verificação de duplicatas por alguns sistemas. Quando se usa o caracter de preenchimento em 008/15-17, as três posições devem ter este caracter.

     

    35-37 — Idioma
    Um có    digo MARC de três caracteres que indica o idioma o ítem. A fonte do código é Lista de Código MARC de Idiomas. A seleção do código MARC se baseia no idioma predominante do ítem. Podem ser usados três caracteres de preenchimento (|||) no caso de não codificar  008/35-37 é preferivel (e codificado no campo 041 (Código da lingua).

     

    38 Modificação do registro

    Recomenda-se usar o valor default

    # - Não modificado

     

    39 — Fonte de catalogação
    Um caracter que indica quem é o criador da catalogação do registro. O nome da organização pode estar no subcampo ^a do campo 040 (Fonte da catalogação).

    # - agência bibliográfica nacional

    d - outra
