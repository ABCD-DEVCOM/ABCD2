Introdução ao MARC 21

O Formato MARC 21 para Dados Bibliográficos visa ser um depositário de informações bibliográficas, tais como títulos, nomes, assuntos, notas, informações de publicação e descrição física dos itens. O Formato MARC 21 para Dados Bibliográficos contem elementos de dados para os seguintes tipos de materiais:

Livros - itens textuais, de natureza monográfica, isto é, livros encadernados, eletrônicos ou microformas.

Seriados - itens textuais com um padrão recorrente de publicação, por exemplo, periódicos, jornais, anuais.

Arquivos de computador - programas de computador, dados numéricos, multimídia em computador, sistemas ou serviços online. Outros tipos de recursos eletrônicos são codificados pelo seu aspecto mais significativo, tal como textual ("livros" ou "séries"), cartográfico ("mapas"), etc..

Mapas - todos os tipos de materiais cartográficos, incluindo mapas planos e globos impressos, manuscritos, eletrônicos, e micro-forma.

Música - partituras musicais impressas ou manuscritas.

Registros sonoros - Registros sonoros não musicais e musicais.

Materiais visuais - imagens e objetos, isto é., mídia projetada, figuras de movimento, gráficos bidimensionais, artefatos tridimensionais, objetos naturais.

Materiais mistos - coleções primordialmente documentos de arquivo e manuscritos de uma mistura de formas de material.

. O "Formato Bibliográfico MARC 21 LITE" é um subconjunto do que está definido no "Formato Bibliográfico Completo MARC 21". Inclui todos os elementos de dados essenciais  necessário para criar descrições bibliográficas de itens de informação. É um subconjunto verdadeiro dos elementos de dados do formato completo do MARC 21 para dados bibliográficos e não altera nenhuma definição dos campos de dados encontrado no formato completo (full). Os registros que usam somente os elementos deste documento são registros MARC válidos e podem ser integrados com registros mais completos sem alteração. Se os elementos do formato completo forem necessários por razões especiais em uma execução, podem sempre ser adicionados aos registros do formato LITE pelo implementador ou pelo usuário.

 

O "Formato Bibliográfico MARC 21 LITE" fornece um guia de referência rápida aos campos. Onde necessário, inclui uma descrição concisa de cada campo, cada posição de caráter dos campos de tamanho fixo do elemento de dados e dos indicadores definidos nos campos de dados variáveis. As descrições de campos com sub-campo e de valores codificados são dadas somente quando seus nomes não forem suficientemente descritivos. Exemplos são incluídos para cada campo.

COMPONENTES DE UM REGISTRO MARC 21

Um registro MARC é composto de três elementos: a estrutura do registro, os campos e outras designações de conteúdo, e o conteúdo dos dados do registro. A estrutura do registro é uma implementação da ISO 2709, Informação e Documentação - Formato para Troca de Informação. A designação de conteúdo é o conjunto dos campos e dos códigos que identificam e caracterizam os elementos de dados dentro de um registro e suporta a manipulação do conteúdo dos dados. O conteúdo dos elementos de dados que compreendem um registro MARC é geralmente definido por padrões externos aos formatos, tais como regras de catalogação, esquemas de classificação, tesauro de assuntos, listas de códigos, ou outras convenções usadas pela organização que cria um registro. O conteúdo de certos elementos de dados codificados (por exemplo, leader, e campo 008) são definidos no próprio formato MARC .

A estrutura do registro MARC consiste em três componentes principais: o leader, o diretório, e os campos variáveis. A informação a seguir resume os componentes de um registro MARC. Mais detalhes são fornecidos em MARC 21 Especificações para Estrutura de Registro, Caracteres, e Media da Troca.

Leader

Elementos de dados que fornecem a informação básica sobre o item e a informação para o processamento do registro. Os elementos de dados contêm números ou valores codificados e são identificados pela posição de caráter relativa. O leader tem um comprimento fixo de 24 posições e é o primeiro campo em todos os registros MARC.

 

Diretório

Uma série de entradas que contêm o Tag (campo), o tamanho, e a posição inicial de cada campo variável dentro de um registro. Cada entrada tem 12 posições de tamanho. A entrada de diretório para o número de controle do registro (001) aparece em primeiro lugar. As entradas subseqüentes para campos de dados variáveis seguem, em ordem ascendente de acordo com o primeiro caráter do Tag. A seqüência armazenada dos campos de dados variáveis em um registro não é necessariamente a mesma que a ordem das entradas  correspondentes no diretório. O diretório termina com um caráter de fim-de-campo (ASCII 1E hex).

 

Campos Variáveis

Os dados em um registro MARC 21 são colocados em campos que podem ser variáveis no tamanho, cada um identificado por um Tag numérico de três caracteres que são constam na entrada de diretório para o campo. Cada campo termina com um caráter de fim-de-campo. O registro MARC termina com um caráter de fim-de-registro (ASCII 1D hex). Há dois tipos de campos de tamanho variável:

Campos variáveis de controle 

Campos 00X. Os campos variáveis de controle são estruturalmente diferentes dos campos de dados variáveis. Estes campos não contêm posições do indicador ou códigos de sub-campos. Podem conter um único elemento de dados ou uma série de elementos de dados de tamanho fixo, identificados pela posição de caráter relativo

 

Campos de dados variáveis

Campos 01X-8XX .

Nos campos de dados variáveis, os seguintes dois tipos da designação de conteúdo são usados:

Posições de Indicador

São as primeiras duas posições de caráter no começo de cada campo de dados variáveis e que contêm os valores que interpretam ou suplementam os dados encontrados no campo. Os valores do indicador são interpretados independentemente, isto é, o significado não é atribuído aos dois indicadores em conjunto. O valor de um indicador pode ser um caráter alfabético em caixa baixa ou numérico. Um espaço em branco (ASCII 20 hex), representado neste documento como um #, é usado no caso de indicador indefinido. Em uma posição definida do indicador, a um espaço em branco pode ser atribuído um significado, ou pode significar nenhuma informação fornecida.

 

Códigos de subcampo

Dois caracteres que precedem cada elemento de dados dentro de um campo que requeira manipulação separada. Um código de subcampo consiste em um delimitador (ASCII 1F hex ), representado neste documento como um $, seguido por um identificador de elemento de dados. Os identificadores de elemento de dados são caracteres alfabéticos caixa baixa ou numéricos. Os códigos dos subcampos são definidos independentemente para cada campo; entretanto, os significados paralelos são preservados onde possível. Os códigos de subcampos são definidos com a finalidades de identificação. A ordem dos subcampos é especificada geralmente por padrões externos para o conteúdo dos dados, tal como as regras de catalogação, não por este formato.

 

Tipo de registro

Um elemento de dados chave do leader é o tipo do item que está sendo descrito no registro. Leader/06 (tipo de registro) identifica os seguintes tipos de itens (materiais).

Repetividade de campos e subcampos

Teoricamente, todos os campos e subcampos podem ser repetitivos. A natureza dos dados, no entanto, muitas vezes impede a repetição. Por exemplo, um registro pode conter somente um campo 1XX; um campo bibliográfico100 pode conter apenas um subcampo $a (Nome pessoal), mas pode conter mais de um subcampo $c (Títulos e outras palavras associadas ao nome). Repetibilidade/não-repetibilidade de campos e subcampos é indicado por (R) ou (NR) seguindo cada nome de campo e subcampo.

Caracter de Preenchimento e Valores Relacionados

Um caráter de preenchimento(ASCII 7C hex), representado neste documento como uma barra vertical (|), pode ser usado em um registro, quando o formato especifica um código para ser usado, mas ocriador do registro decide não tentar fornecer um código. Um caráter de preenchimento não deve ser usado em qualquer posição do LEADER, TAG's, indicadores ou códigos de subcampos.

Código u (Desconhecido ou não especificado), quando for definido é usado para indicar que o criador do registro tentou atribuir um código, mas não pode determinar qual deveria ser o código apropriado.

Código n (Não aplicável), se for definido é usado para indicar que a característica definida pela posição não é aplicável para um tipo de ítem ou registro.

Constantes de Apresentação

Constante de apresentação é um termo, frase e/ou convenção de espaçamento ou pontuação que pode ser gerado pelo sistema para tornar a apresentação visual de dados em um registro mais significativa para o usuário. O texto de apresentação não é informado no registro, mas há uma indicação para o mesmo, baseado em TAG´s, indicadores, subcampos, ou valores codificados. Sugestões de constantes e exemplos de apresentação são oferecidas. O uso e apresentação destas constantes são determinados por cada organização ou sistema.

CONTEÚDO DO REGISTRO

Responsabilidade

Em geral, a responsabilidade pelo conteúdo dos dados, a designação de conteúdo, e a transcrição dos dados dentro de um registro MARC 21 podem ser determinadas pela verificação do campo indicado no campo 040 (agência de catalogação original).

Em registros não modificados, a organização identificada como a fonte catalogadora original no 040 $a (agência de catalogação original) é responsável pelo conteúdo dos dados do registro. As organizações identificadas como a agência de transcrição no subcampo $c do campo 040 (agência que transcreveu) são responsáveis pela etiquetagem (taging) e a transcrição dos dados.       

Em registros modificados, as organizações identificadas no campo 040 $a (agência catalogadora original) e $d (agência modificadora) são coletivamente responsáveis pelo conteúdo dos dados do registro. As organizações identificadas como agências de transcrição ou modificadoras no campo 040 $c e $d são coletivamente responsáveis pela etiquetagem e a transcrição dos dados.

Um elemento de dados atribuído por agência é aquele cujo conteúdo é determinado por uma agência designada e é da responsabilidade dessa agência, por exemplo, o campo 222 (título chave) que é de responsabilidade de um centro de ISSN. Quando geralmente é atribuído pela agência designada, pode ser transcrito por uma outra organização.

Determinados elementos de dados contêm os dados das listas controladas, mantidas por agências designadas, por exemplo, a lista de códigos do MARC para línguas no campo 041 (Lista de Código MARC de Idiomas )  do formato bibliográfico. Estes elementos são indicados a nível de campo ou subcampo no MARC 21 e somente os valores das listas designadas podem ser usados. Se uma mudança ou acréscimo forem desejável para uma lista, a agência mantenedora da lista deve ser consultada.

Registros Multi-alfabetos (Multiscript)

Um registro MARC 21 pode conter dados em múltiplos "scripts". Um "script" pode ser considerado pelo "script" preliminar do conteúdo de dados do registro, mesmo que outros "scripts" também sejam usados para conteúdo de dados. (Nota: ASCII é usado para elementos de estrutura do registro, com a maioria dos dados codificados também especificados no intervalo de caracteres ASCII.). Exemplos e modelos para dados multi-alfabetos são descritos na seção Registros Multiscript.

Convenções Tipográficas

Neste documento as seguintes convenções tipográficas são usadas:

0 - O gráfico 0 representa o dígito zero em tag´s, citações de posição de caráter de posição fixa, e posições de indicador. Este caráter é distinto da letra O (caixa alta), usado em exemplos ou texto.

# - O símbolo gráfico # é usado para representar um branco (hex 20) em campos codificados e em outras situações especiais onde a existência de um caráter branco pode ser ambíguo.

$ - O símbolo gráfico $ é usado para parte do delimitador (hex 1F) em um código de subcampo. Em um texto, códigos de subcampos são indicados como  subcampo $a , por exemplo.

/ - Posições específicos de caráter de elementos de dados de tamanho fixo, tais como no Leader, Diretório, e campo 008, são expressos usando uma barra e o número da posição, ex. Leader/06.  

1 - O gráfico 1 representa o dígito um (hex 31). Este caracter deve ser distinto da letra  l (ele) do alfabeto romano em caixa baixa (hex 6C) em exemplos ou texto.

| - O gráfico | representa um caráter de preenchimento (hex 7C).
