Configurar la base de datos de usuarios.

Esta opción permite configurarle al sistema la base de datos con los usuarios que tienen acceso al servicio de préstamos.

Los parámetros requeridos son:

Prefijo para obtener el número de usuario Especifica el prefijo que se antepone al número del usuario introducido a fin de localizar la información en la base de datos de usuarios. Se almacena en la carpeta def de la base de datos de usuarios bajo el nombre loans_uskey.tab
PFT para extraer el número de usuario Una vez localizado el registro del usuario, se aplica este formato de extracción para obtener el número del usuario.Se almacena en la carpeta def de la base de datos de usuarios bajo el nombre loans_uskey.pft
PFT para extraer el tipo de usuario Las políticas de préstamo se aplican dependiendo del tipo de usuario y el tipo de objeto a prestar. Este formato de extracción permite extraer el tipo de usuario. Se almacena en la carpeta def de la base de datos de usuarios bajo el nombre loans_ustype.tab
PFT para extraer la vigencia del usuario

Mediante este formato se establece la vigencia del usuario. Debe contemplar un mensaje y un código, separados ambos por el caracter |.
Los códigos establecidos son:

0=Usuario activo
1=Usuario no activo

Se almacena en la carpeta def de la base de datos de usuarios bajo el nombre loans_usvig.tab
PFT para desplegar la información del usuario Este formato de extracción presenta la información del usuario para la emisión de estados de cuenta y otros reportes relacionados con la gestión de los préstamos. Se almacena en la carpeta def de la base de datos de usuarios bajo el nombre loans_usdisp.tab
