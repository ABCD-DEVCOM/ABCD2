Este módulo trabaja con las siguientes bases de datos:

users
Base de datos de usuarios
loanobjects
Base de datos con las diferentes copias de los objetos que están disponibles para préstamo.

Existe una sola base de datos loanobjects que refleja las existencias de todos los catálogos definidos bajo ABCD. Cada copia se identifica a través de un número único o número de inventario. Almacena además el número de control del objeto, el volúmen y el tomo de la copia a efectos de establecer las diferencias entre las copias de un mismo objeto;  el nombre de la base de datos donde está catalogado, a efectos de identificación de la copia;  y el tipo de objeto asignado a la copia a efectos de aplicación de la política de préstamos.

Cada objeto representa un registro en la base de datos loanobjects y las copias del mismo se almacenan como ocurrencias de un campo repetible.

Esta base de datos se actualiza desde el procedimiento de catalogación donde se permite transferir los registros de inventario desde la base de datos copies hacia la base de datos loanobjects
Base de datos de catálogo (cepal, marc, etc)
Descripción bibliográfica de los objetos disponibles para préstamo. El vínculo entre el catálogo y sus existencias se realiza a través del número de control del objeto
trans
Transacciones de préstamos, renovaciones y devoluciones
suspml
Suspensiones y multas registradas a los usuarios
reserva 
Reservas

El menú principal del menú del módulo de Circulación y Préstamos contiene la siguientes secciones:

Transacciones

Esta sección define las transacciones disponibles:
  • Prestar
    A través el código del usuario y del número de inventario del objeto se acceden las bases de datos users y loanobjects a fin de determinar el tipo de usuario y el tipo de objeto. Se verifica además si el usuario tiene pendientes multas y/o suspensiones realizando una búsqueda sobre la base de datos suspml, o si tiene préstamos pendientes, realizando una búsqueda sobre la base de datos trans. Con estos valores se determina la política de préstamos a aplicar y se calcula la fecha de devolución
  • Devolver
    Se utiliza el número de inventario para localizar en la base de datos trans la copia que se desea devolver. Si hay atraso en la fecha de devolución se calcula la sanción que indique la política y la misma se guarda en la base de datos suspml
  • Renovar
    Se utiliza el número de inventario para localizar en la base de datos trans la copia que se desea renovar.     Se verifica además si el usuario tiene pendientes multas y/o suspensiones realizando una búsqueda sobre la base de datos suspml, o si tiene otros préstamos pendientes, realizando una búsqueda sobre la base de datos trans. Con estos valores se determina la política de renovación a aplicar y se calcula la nueva fecha de devolución
  • Suspender/Multar
    Se suministra el código del usuario que se desea suspender/multar y los datos correspondientes.
  • Estado de cuenta de un usuario
    Presenta la lista de préstamos, suspensiones y/o multas pendientes para un usuario
  • Situación de un objeto
    Presenta todas las copias de un objeto, indicando aquellas que están en poder de algún usuario
