Bloqueo y desbloqueo de registros

Cuando a un operador se le concede la edición de un registro, el mismo se bloquea para que otro usuario no pueda actualizarlo en forma simultánea.

Cuando este registro es actualizado o se cancela su edición (botones guardar o cancelar de la barra de herramientas), el registro se libera quedando disponible para el resto de los operadores

Si el ciclo de actualización no se cumple, esto es, el registro no se escribe ni se cancela la actualización (botones guardar o cancelar de la barra de herramientas), el registro permanece bloqueado por un tiempo (4 horas) para el resto de los usuarios.

Existen dos formas para eliminar el bloqueo antes de que se cumpla el período antes mencionado:

  1. Si el mismo operador que bloqueó el registro, solicita de nuevo su edición, entonces, se le concede el derecho a actualizar el registro, pudiendo actualizar o cancelar la edición para liberar el registro
  2. Otra forma de desbloquear los registros es utilizando la opción "Desbloquear registros" contenido en el menú Utilitarios de la barra de herramientas. Solo el Administrador del Sistema o el Administrador de la Base de Datos tienen acceso a esta opción.

Recuerde que el formato de presentación de los términos de la lista controlada puede ser diferente al formato de extracción del campo, esto es, el formato de presentación puede omitir los subcampos, pero el formato de extracción puede incluir los subcampos. Validación de registros

La validación de registros permite definir para cada campo de la base de datos, un formato (PFT) que se utilizará para determinar la consistencia de los datosantes de almacenar cada registro  o al hacer clic sobre el vínculo Validar registro cuando se está haciendo el recorrido de la base de datos. 

El formulario de validación contiene una entrada para cada campo definido en la FDT y una ventana de texto para colocar el formato de validación a ser aplicado individualmente sobre cada campo.  Además se presenta una casilla de selección () la cual permite condicionar la grabación del registro al resultado de la validación: si la casilla está marcada y se está ingresando o modificando un registro, entonces el registro no será grabado si alguno de los campos marcados no supera la condición de validación. La condición de validación se considera no superada si el formato que se aplica sobre el campo devuelve algún valor.

En el procedimiento de ingreso, al momento de actualizar un registro, se construye un formato donde se consolidan las validaciones parciales. Este formato  se aplica sobre el registro y a través del mismo se puede detectar cuál campo generó error y determinar si el error producido permite o no la actualización del registro.

Si el registro no supera la validación, se muestran los errores encontrados. Si  alguno de los campos que generan el error están marcados en el formato de validación como Error fatal (seleccionando la casilla correspondiente en el formato de validación), el registro no se actualiza y se presenta un vínculo para retomar los cambios realizados y efectuar las correcciones necesarias.

Si  ninguno de los campos que generan error están marcados como Error fatal se emite el reporte de validación y el registro de actualiza.

   para cancelar la edición del registro, o

  para eliminar el registro

Nota importante: Si el registro no sufrió ningún cambio es conveniente utilizar el botón cancelar en lugar de actualizar, para garantizar la integridad de la base de datos
