Agregar copias a un objeto

Los objetos a los cuales se le inventarían las copias deben poseer un campo denominado "Número de control" el cual debe estar definido en la FDT con el tipo "autoincrement" y estar indizado en la FST con el prefijo "NC_".  Este campo de la FDT se utiliza como campo de relación entre un objeto y sus copias.

Las copias de los objetos todas las bases de datos confluyen en una base de datos denominada copies y se mantiene un único número de inventario para cada copia.

Para la asignación secuencial del número de control en cada registro, se utiliza el archivo control_number.cn ubicado en la carpeta data de la base de datos correspondiente. Cada base de datos bibliográfica posee su propia secuencia del número de control. Por esta razón, en la base de datos copies se almacena el nombre de la  base de datos relacionada con la copia para garantizar la correcta recuperación de la información bibliográfica, cuando ello sea necesario.

Cuando se solicita el ingreso de nuevas copias, se presenta un formulario para ingreso de la información correspondiente.

Coloque en el recuadro el número de copias que desea agregar al objeto seleccionado. Use el vínculo Ver todas las copias para desplegar todas las copias ya ingresadas para ese objeto.

Rellene los datos correspondientes y cuando termine haga clic sobre Actualizar.

Se creará en la base de datos copies el número de registros solicitados, de acuerdo al valor Número de copias. La información de los registros creados se tomará del formulario de ingreso. El número de inventario se incrementará automáticamente tomando como punto de partida el valor almacenado en al archivos control_number.cn almacenado en la carpeta bases/copies/data.

Use la opción "Restablecer el número de control o de inventario" del menú principal de catalogación, si por alguna razón se necesita restaurar el próximo número de inventario a asignar a las copias, o a los objetos de una base de datos en particular.
