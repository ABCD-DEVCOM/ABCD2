1. Display  information de l'objet obtenu à partir du catalogue de la base de données. Pour déterminer le préfixe à appliquer dans cette recherche il faut lire la TDC du catalogue de la base de données et trouver le champ type AI, qui doit indiquer le code utilisé pour indexer le numéro de contrôle de l'objet. La valeur du préfixe associée à l'identification de l'objet mise sur le bon de commande fournit l'expression de recherche à appliquer à l'emplacement de l'objet.
  2. Il fait une recherche dans les COPIES de la base de données pour vérifier si les objets de ce bon de commande ont été mis à jour. Il construit une expression de recherche avec les champs suivant :
    ORDER_order_suggestion_bidding

    where  order=number  of purchase order,  suggestion=recommendation number, bidding=quotated number.
    Cette recherche va essayer d'empêcher la duplication dans les copies de la base de données, produite par la perte de connexion ou d'autres facteurs.
    Si l'information des copies de la base de données est localisée pour mettre en oeuvre la recherche spécifiée, ayez à l'esprit cette condition et ne permettez pas la création de copies. Un message comprenant un lien est fournit pour rendre disponible des copies dans la base de données copies
  3. Si aucune information n'est trouvée dans les copies de la base de données, la création de données dans la base de données est faite en utilisant les valeurs citées sur le bon de commande. 
  4.  on procède à la création des enregistrements dans cette base de données en utilisant les valeurs figurant dans le bon de commande. La valeur nombre de copies indique le nombre de données dont vous avez besoin pour créer des copies de la base de données.

    A chaque donnée est attribuée un numéro séquentiel correspondant au numéro d'inventaire.  
    Pour cette raison, il faut un fichier  control_number.cn localisé dansdonnée dossier de la base de donnéescopies pour sauvegarder le dernier numéro d'inventaire attribué.
  5. Dans la FDT descopies de la base de données il y a nécessairement un fichier type AI (autoincrement) pour conserver a  le numéro d'inventaire généré.

    L'information minimale contenue dans les copies de la base de données se présente comme suit :

    40 Type d'Objet

  6. Après avoir créé des copies de l'objet et attribuer  un numéro d'inventaire changez pour permettre au buton de continuer de créer les données correspondantes dans la base de données des objets de  prêt (objets de prêt)
