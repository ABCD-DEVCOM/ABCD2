Ce module fonctionne avec les bases de données suivantes:

utilisateurs
Base de données utilisateurs
loanobjects
Base de données avec différentes copies d'objets sont disponibles pour emprunt.

Il y a une seue base de données loanobjects qui reflète le stock de tous les catalogues définis à l' ABCD. Chaque exemplaire est identifié par un numéro unique ou numéro d'inventaire. Aussi le numéro de contrôle de l'objet, le volume et le tome de la copie aux fins d'établir les différences entre les copies d'un même objet;  le nom de la base de données est cataloguée à des fins d'identification de la copie;  et le type de l'objet assigné à la copie aux fins de la politique de prêts.

Chaque objet représente une ligne dans la base de données loanobjects et les copies de ce loanobjects peuvent être stockées en tant que les occurrences d'un champ répétitif.

Cette base de données est mise à jour du processus de catalogage qui permet de transférer des registres d'inventaire de la base de données copies à la base de données loanobjects

Transactions

Ce chapitre définit les opérations disponibles:
  • Le prêt
    A travers le code de l'utilisateur et le numéro d'inventaire de l'objet sont accessibles les bases de données utilisateurs et loanobjects pour déterminer le type d'utilisateur et le type d'objet. Il vérifie également si l'utilisateur a des aamendes et/ou suspensions par la base de données suspml, ou si vous avez des prêts, par la recherche de la base de données trans. Ces valeurs déterminent la politique de prêts à mettre en oeuvre et on estime la date de retour
  • Retour
    Nous utilisons le numéro d'inventaire pour localiser dans la base de données trans l'exemplaire à retourner. S'il y a du retard dans la date de retour de la peine est calculée à indiquer la politique et il est stocké dans la base de données suspml
  • Renouveler
    Nous utilisons le numéro d'inventaire pour localiser dans la base de données trans l'exemplaire que vous souhaitez renouveler.     Il vérifie également si l'utilisateur a des amendes et/ou de suspensions faisant une recherche dans la base de données suspml, ou si vous avez d'autres que des prêts, par la recherche de la base de données trans. Ces valeurs déterminent la politique de renouvellement et de mettre en oeuvre la nouvelle date de retour estimé
  • Suspends/Amendes
    Il prévoit le code de l'utilisateur soit que vous voulez suspendre/ou mettre en amende ainsi que les données correspondants.
  • Etat de comptes des utilisateurs
    La liste des suspensions de prêt et/ou des amendes non payées d'un utilisateur
  • Location d'un objet
    Présente tous les exemplaires d'un objet, en précisant ceux qui sont détenus par un utilisateur
