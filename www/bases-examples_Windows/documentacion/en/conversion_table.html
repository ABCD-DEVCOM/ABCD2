Conversion table for exporting the database to a TXT file with tags

The conversion table is used both to export and import records, applying in each case the following features:

Use the box Delimited with Tab to import an Excel sheet saved as TXT. In this case the column Tag in the TXT file must contain the number of the column (starting with 1) where the corresponding field is located in the exported Excel sheet.

The conversion table contains the following columns:


Tag in the TXT file

Tag which identifies the field in the input or output TXT file. The tags have to start with $$ and end with :

Tag

Tag of the field in the database

Subfields

Subfield delimiters. If the first subfield doesn't have any delimiter, leave a blank space

Replace by

If a field has subfields, indicate the characters which will be used in the text to replace the subfield marks here. The first character of this list would replace the first subfield in the list of subfields. If the first subfield is not going to be replaced by any delimiter, leave a blank space

Occurrences separator

Indicate the character which will separate the occurrences of a repeatable field here

Extraction format

This format operates on the record to produce the output of the associated field. The application of the extraction format is only executed during the exporting process. If you provide subfield delimiters plus the extraction format, the last one is the one to be applied in the process (not implemented yet).
