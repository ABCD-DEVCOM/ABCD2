Editing the file DBN.def to configure a database to run with the iAH OPAC 

This configuration page contains 4 mandatory sections to configure a database for IAH:

  • FILE_LOCATION
  • INDEX_DEFINITION
  • APPLY_GIZMO
  • FORMAT_NAME
  • HELP_FORM
  • PREFERENCES

For information on these sections, please consult the IAH User Manual. Here we provide help on how to use the configuration utility in ABCD but not with the content of the sections.

To edit the configuration file there are 2 options:

  1. Use a TXT-editor to modify the configuration file. For this click on the link Edit TXT format at the top of the page.
  2. Use the guide provided by ABCD to facilitate filling in the sections.

The guide is divided into the 6 required sections. Each one has a link add at the end to allow addition of a new element in the section being edited.

The section   INDEX_DEFINITION allows each line to be edited in order to fill in the information. For this click on the link edit which accompanies each index definition and which will put the corresponding information in the form. Once the form is completed, click on Accept to transfer the cells to the corresponding line of the section INDEX_DEFINITION.

The section FORMAT_NAME  allows access to the PFT format editor of ABCD for creating or editing the corresponding format.  This can be accessed by clicking on the link Create/edit format at the top of the section. When creating a new format, fill in the in the cells of the section providing the information shown when clicking on Add.

Once the form is completed, click on Validate in order to verify the consistency of the information entered and then on Update to save the newly created file.

The validation criteria used are the following :

FILE_LOCATION

  1. Verify that all logical names are preceded by the literal FILE. If missing, add.
  2. Verify that all paths are preceded by the literal %path_database%. If missing, add.

INDEX_DEFINITION

  1. Verify that all logical names are preceded by the literal INDEX. If missing, add.
  2. Verify that at least one subfield contains the name of the index.
  3. Verify that the logical names of databases of the indexes (^y) have ther corresponding entry in the section FILE_LOCATION

APPLY_GIZMO

  1. Verify that the names of the gizmo files provided have their corresponding entry in the section FILE_LOCATION

PREFERENCES

  1. Verify that at least one search form is active.
  2. Verify that the number of documents per page is a numerical value.
