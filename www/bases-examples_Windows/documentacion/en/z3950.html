Copying records from remote servers using the Z39.50 protocol

This option in ABCD permits cataloguing directly from other sources available through the Z39.50 protocol.

Under the menu  Conectar a:  a list of servers which are available by the location of servers is shown. This list is maintained through the option Configure Z39.50 client available (to the administrator) in the principal menu of ABCD.

The menu Conversion Table contains the schemes which have been defined for converting records from the MARC format  to the field definitions in the local database that will receive the data. This list is maintained through the option Configure Z39.50 client available (to the administrator) in the principal menu of ABCD. If no conversion scheme is selected, the FDT of the database is used to determine the fields to copy from the remote database.

Use the search options to establish the criteria to apply for customizing the information that you want to catalogue.

In the box Show indicate how many criteria are to be considered for configuring the pages showing the results obtained.

Try xx times to connect specifies how many times the system attempts to make a connection when the server cannot be reached.

In normal conditions, the records transferred create new records in the local database that receives the data.  To transfer the data to a record that already exists, follow the procedure below:

  1. In the local database locate the existing record that should receive the catalogue data through Z39.50.
  2. After locating the record, click the button edit on the toolbar. 
  3. When the form for editing the record is displayed, click the button that calls the Z39.50 client. If the record is being modified, you should save the record before calling the Z39.50 client since the other form will lose the modifications made.
  4. Next select the database from which to catalogue, the conversion format (if necessary) and formulate the search.
  5. In the results presented, locate the record you wish to capture.
  6. Click on the Capture icon. The data retrieved from the Z39.50 server is copied into the selected record:
    • Converting the data by applying the conversion format (if selected)
    • Transferring the data to the fields of the local database, only when the corresponding ones are empty, i.e. the action of copying does not replace data already existing in the record.
  7. Save the record in the local database.
