Uploading an ISO file

The ISO files that are created in the export operation are stored in the folder ./bases/wrk. To import an ISO file click on the option Select that appears with the ISO file that you want to import.

If you want to substitute the whole database with the imported records, tick the box called Delete all the records before importing. If you don't tick this box, all the imported records will be loaded after the last record in the database.

If you want to update the inverted file while the import is being done, tick the box called Create the inverted file. If you don't select this option, you can activate the creation of the inverted file later on using the option for it in the Maintenance menu of the database.

Important note.: If the database has many records, it is advisable to use the MX utility for the creation of the inverted file.

If the ISO file that you want to import is not on the server, you would need to upload it first before proceeding to importing. For that purpose, use the section Upload ISO file that is shown in the import window. Once the file is transferred, you can import it.

Important note.: the ISO file to import has to have the same tags and subfields defined in the database because importing an FST to make changes to the structure of the fields is not available.

If the ISO file to upload is big, you need to use a FTP or other means to upload it to the server.

Use the option Delete that comes with each ISO file shown, in order to delete those ISO files that have been already used from the server.
