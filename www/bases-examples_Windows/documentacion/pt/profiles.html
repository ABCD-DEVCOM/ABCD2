As permições de ABCD se agrupam abaixo das categorias:

1. Permições sobre as bases de dados e formatos de entrada e saída definidas

2. Permições sobre as funcionalidades do ABCD.

Para proporcionar permições aos usuários se definem perfis que fazem referência tanto as bases de dados como as funcionalidades às cuais tem acesso os usuários pertencentes a cada perfil.

 Criação de perfis

Use a opção Criar/editar perfis para definir as permissões a atribuir aos usuários que se agrupam esse perfil.

Um perfil pode conter tantas bases de dados como seja necessário e possuir acesso a um ou mais módulos.  nste último caso se apresentará no menu principal um menu para que o operador possa mover-se através dos módulos selecionados.

Uma vez que construir perfis, deve ingressar os usuários e lhes atribuir o perfil corresponde.

Os perfis podem editar-se ou eliminar-se uma vez que hajam sido criados. No entanto, se um perfil está atribuído a um usuário o mesmo não poderá eliminar-se até que exclua as relações dos usuários que se faziam a este esse perfil.

ABCD possui um grupo denominado adm o qual corresponde ao administrador do sistema. Não é permitido editar ou excluir o perfil deste grupo e pelo menos um usuário deve estar atribuído ao mesmo. De outra forma ocorre o risco de desabilitar o acesso funções críticas do sistema.

Se por engano o administrador do sistemafor removido, siga o seguinte procedimento para atribuir um administrador provisório para entrar a ABCD e definir o novo administrador:

  1. Edite o arquivo config.central presente na pasta central e coloque um valor provisório nos parâmetros $adm_login e  $adm_password. Este valor provisório não deve corresponder a nenhuma das identificações de usuário já existentes na base de dados de controle de acesso

  2. Entre a ABCD e utilize como identificação do usuário e palavra chave os valores inseridos em $adm_login e  $adm_password, respectivamente

  3. ABCD lhe proverá acesso com o perfil do administrador do sistema. Veja então à opção Administração de usuários e restituía ao administrador do sistema.

  4. Apague config.central

Os perfis de usuário se armazenam na pasta par/profiles. Nela existe um arquivo para cada perfil criado. O arquivo profiles.tab  é interno a ABCD e provê os parâmetros que podem utilizar-se para restringir/acessar as funcionalidades do ABCD. Não deve apagar já que é a matriz utilizada para a criação/edição de perfiles.


O arquivo adm contém o perfil do administrador do sistema e não pode editar-se nem apagar-se. Por esta razão não se apresenta na lista de perfis ativos.
