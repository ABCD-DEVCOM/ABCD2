Resultado da criação de uma nova base de dados

Neste procedimento é criada a estrutura para armazenar a nova base de dados na pasta bases.

As pastas criadas neste processo são as seguintes:

  • bases/dbn/ayudas
  • bases/dbn/data
  • bases/dbn/def
  • bases/dbn/pfts
  • bases/dbn/cnv

Sob cada uma destas pastas, exceto as pastas data e CNV, são criadas tantas subpastas quantos os idiomas ativos. A lista dos idiomas é obtida a partir do arquivo lang.tab localizado na pasta bases.

Adicionalmente é criado o arquivo dbn.par na pasta bases/par e é atualizado o arquivo bases.dat acrescentado da base de dados criada.

Depois de terminado o processo de criação, deve-se definir os operadores que podem acessar a nova base de dados, porque do contrário só o Administrador do sistema terá acesso à mesma.

Para carregar os dados na base deve-se obter um arquivo ISO com os dados a serem importados para a nova estrutura.

Se o arquivo vem de WinIsis ou IsisDos, será necessário converter o conjunto de caracteres de ASCII para ANSI. Esta tarefa pode ser realizada facilmente utilizando o utilitário MX da biblioteca de funções CISIS.

Para isto,  em uma janela DOS, execute o MX a partir da pasta onde está localizada a base de dados que deseja converter, utilizando a seguinte línha de comandos:

mx dbn  iso=dbn.iso convert=ansi -all now tell=100

O arquivo iso obtido deve ser transferido para o servidor, pasta data da base de dados em processo de criação. Para incorporar os registros use também o programa MX, com a seguinte linha de comandos:

mx iso=dbn.iso create=dbn -all now tell=100.

Depois de criar a base de dados, execute o MX novamente para gerar o arquivo invertido, utilizando a seguinte linha de comandos:

mx dbn fst=@dbn.fst uctab=ansi actab=ansi fullinv=dbn  -all now tell=100

Nota: Os comandos Mx aqui apresentados servem apenas como orientação. Para mais informações deve-se consultar o manual de referência da biblioteca CISIS.

Se a número de registros a importar não for muito grande (por exemplo, menos de 2000 registros)  podem ser utilizadas as opções Utilitários da barra de ferramentas do procedimento de Entrada de dados para importar registros e gerar o arquivo invertido.

Como o tempo configurado no servidor Web para cada processo depende de cada instalação, o número de registros aqui mencionado pode variar, por isso se recomenda sempre que possível usar o MX para estas tarefas.
