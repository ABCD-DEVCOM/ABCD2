Atribuição de um número de controle aos registros de uma base de dados


Esta opção permite atribuir de forma automática um número sequencial consecutivo, aos registros de uma base de dados.

Na FDT deve definir o campo de controle, como tipo de campo "Autoincrement". Se vai ser utilizado para o processo de inventario através do sistema de aquisições, deve indicar o prefixo NC_ na FST.

A numeração é gravada no arquivo control_number.cn, armazenado na pasta data da base de dados correspondente. Neste arquivo é gravado o último número atribuído no processo de entrada dos registros. Para modificar a sequencia de atribuição deste número, use a opção Restabelecer número de inventário ou número de controle contida na página de inicio do módulo de catalogação.

Quando se solicita a atribuição automática do número de controle, é exibida uma janela onde se coloca a faixa de Mfns aos quais será atribuído este número. Tenha em conta que se coloca uma faixa de mfn's demasiado ampla, o tempo empregado para a atribuição dos números pode causar um "time out" no servidor e o processo pode ser interrompido.

Neste caso, revise a saída exibida, revise o último mfn processado e reinicie o processo a partir do próximo MFN.

Dado que este processo atualiza todos os registros da faixa especificada, é recomendado suspender todos os processos de entrada e ediçãode registros até que o mesmo tenha sido completado.

Recomenda-se reorganizar a base de dados em seguida, já que o tamanho em bytes da mesma pode ser duplicado, produto da atualização dos registros. Igualmente, deve-se gerar o arquivo invertido.
