Apresenta a ordem emitida, especificando a lista de objetos que a compoem. Para cada objeto se mostram os dados de:
  • Referencia do objeto
  • Número de exemplares adquiridos
  • Preço unitário de cada exemplar
  • Número da sugestão onde se solicitou a aquisição do objeto
  • Referencia da licitação que generou a ordem de compra
  • Base de dados onde se catalogará (novo) ou está catalogado (exemplares) o objeto
  • Tipo de adquisição: novo ou copias
  • Identificação do objeto, no caso de que o tipo de adquisição corresponda a copias
Esta informação se obtém aplicando o formato receiving.pft sobre o registro localizado na base de dados purchaseorder

Se necessário, se modificam os valorer correspondentes ao número de exemplares e preço unitário. Clique em continuar para processar o objeto.

Quando se constrói este formato se faz uma busca na base de dados catalográfica naqual pertenece o objeto, utilizando como chave o Id. do objeto inserido na ordem de compra. Para determinar se o objeto existe, o formato receiving.pft inclui o seguinte comando:

if s(v500^d)<>'' then f(npost([v500^d],'NC_'v500^c),1,0) fi/fi)

Isto é , se o objeto que se está processando inclui a informação da base de dados catalográfica e o número de controle do registro noqual pertenecem os exemplares adquiridos, então, se localizam os "postings" correspondentes na base de dados catalográfica e se o resultado não é certo, então, se assume que já existe e se fornece um link para que possa ser revisado antes de continuar com o processo.
