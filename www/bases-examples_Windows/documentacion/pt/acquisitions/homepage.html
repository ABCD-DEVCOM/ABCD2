O módulo de aquisição se apoia em quatro bases de dados:

  1. Sugestões Contem as solicitações dos usuários quanto a necessidade de adquisição de novos objetos, ou novos exemplares de objetos já existentes. Contém 4 grupos de informações:
    • Dados do objeto solicitado
    • Dados da pessoa ou departamento que solicita o objeto
    • Dados da aprovação ou recusa da solicitação
    • Dados das licitação é para a compra do objeto
      • Fornecedor
      • Número da licitação
      • Quantidade
      • Preço unitário
      • Moeda
      • Dados da aceitação da licitação.
         
  2. Ordem de compra
    Armazena a ordem de compra emitida como parte do processo de aquisições. Contém :
    • Fornecedor
    • Número da ordem de compra
    • Data da ordem de compra
    • Dados do objeto adquirido
      • Número da requisição
      • Número da licitação
      • Quantidade
      • Preço unitário
    • Reclamações
  3. Fornecedores Com a informação dos fornecedores que podem participar nos processos de licitação e compra dos objetos
  4. Exemplares
    Contém cada uma dos exemplares que foram adquiridos como parte do processo de aquisições.

O processo a seguir é o seguinte:

  1. Se cria uma nova solicitação de compra de objeto na base de dados suggestions. Esta solicitação pode referir-se a adquisição de um novo objeto ou a aquisição de novos exemplares de um objeto existente.

    Para cada registro que ingresa na base de dados suggestions se cria en forma automática um campo que posee um número sequencial o cual se utilizará posteriormente como campo de relação entre as sugestão é e as ordem é de compra generadas

    Cada objeto na base de dados catalográfica se identifica com um número único que servirá posteriormente a establecer a relación entre o objeto e sus copias.

    Cada exemplar também adquirá um número de inventário, noqual se utilizará posteriormente no módulo de circulação.
    se o objeto já existe na base de dados, então a solicitud deve indicar o número asignado a ese objeto na base de dados catalográfica a a cual pertenece. se é um novo objeto, então, se insirá o número ao momento de catalogação do objeto, uma vez finalizada a compra.
  2. Cada solicitação deverá ser aprovada ou recusada. Se é aprovada será necessario indicar o número de exemplares que vão a ser adquiridos
  3. As solicitações aprovadas se somam a um processo de licitação, anotando no registro:
    • Fornecedores
    • Número da licitação
    • data da licitação
    • Quantidade ofertada
    • Preço unitário
    • Moeda
  4. Com a informação da licitação é tomada a decisão de compra e atualiza a base de dados sugestões (suggestions) a informação do fornecedor selecionado:
    • Data da aprovação 
    • Opcionalmente se atualizam os campos de quantidade de exemplares e preço unitário
  5. Se emitem as ordem é de compra com os fornecedor é seleccionados. Estas ordensé de compra armazenam na base de dados purchaseorder e se vinculam a base de dados  suggestions mediante ao número sequencial criado no momento de ingresso da sugestão
  6. Se há atrasos na chegada do material, então se anota na base de dados de ordem de compra a data dos diferentes reinvindicações realizadas.
  7. Quando chega o material  adquirido, existem duas opções :
    • Se o objeto for novo, se deve inserir o número de identificação do objeto, catalogá-lo e criar, na base de  dados copies ,um registro para cada exemplar, inserindo o seu número de inventário
    • Se o objeto já existe na base de dados catalográfica, só se agregará um novo registro na base de dados copies com o número de inventário correspondente

Transações

Sugestões 
Use esta opção para inserir na base de dados as solicitações para compra de novos objetos. Abaixo esta opção poderá criar novas solicitações e indicar qual são aprovadas ou recusadas, colocar a licitação presentada pelos fornecedores e indicar o fornecedor cuja licitação foi aprovada

Ordem de compra 
Esta opção permitirá:

  • Gerar a ordem de compra para os fornecedores cuja licitação foram aprovadas
  • Inserir na base de dados de sugestões os dados da ordem de compra elaborada para a adquisição do objeto solicitado
  • Revisar as ordens de compra pendente e inserir as datas das reclamações
  • Eliminar ordem é de compra e modificando o estado da sugestão associada
  • Processar a chegada de os objetos adquiridos:
    • Realizar a pré-catalogação (si são novos objetos)
    • Gerar os registros correspondentes aos exemplares adquiridos
    • Disponibilizar os exemplares nosistema de circulação e empréstimo
